Comparacion de microcontroladores

Brian Carl Simpson y Reyes & Hector Ojeda Serafio
Ingenieria Mectronica
Fecha: 02 08, 2025
Arduino Attiny Esp32 Electrónica

- Resumen -

En esta práctica se realizó una comparación entre los microcontroladores Arduino Uno, ESP32 y ATtiny ejecutando cuatro códigos idénticos en simulación y en hardware físico. Se evaluó su desempeño en términos de velocidad de ejecución, consumo energético, facilidad de programación y compatibilidad con diferentes entornos de desarrollo.

- Introducción -

Los microcontroladores son la base de numerosos sistemas embebidos, utilizados en aplicaciones que van desde la automatización hasta el Internet de las Cosas (IoT). Para determinar cuál es más adecuado en diferentes situaciones, es fundamental comparar su rendimiento bajo las mismas condiciones.

En esta práctica se evaluará el desempeño del Arduino Uno, ESP32 y ATtiny mediante la ejecución de cuatro códigos idénticos. Se analizarán aspectos como velocidad de procesamiento, consumo energético y facilidad de implementación, tanto en simulación con Wokwi como en pruebas con hardware real

Con este análisis, se pretende identificar las ventajas y limitaciones de cada microcontrolador, permitiendo seleccionar la mejor opción según los requerimientos específicos de un proyecto.

- Materiales -

Microcontroladores: Arduino Uno, ESP32 y ATtiny.

LEDs: 1 por cada microcontrolador..

Resistencias: 220Ω para limitar la corriente de los LEDs.

Botones pulsadores: Para las pruebas de control con entrada digital.

Resistencias de pull-down: 10kΩ para los botones.

Protoboard: Para conexiones temporales.

Cables jumper: Macho-macho y macho-hembra.

Computadora: Para programación y simulación.

Software de simulación: Wokwi..

Cables USB: Para la conexión y programación de Arduino y ESP32.

- Desarrollo -

En esta práctica se compararon los microcontroladores Arduino Uno, ESP32 y ATtiny ejecutando cuatro códigos básicos en simulación y hardware físico. Los programas consistieron en encender un LED, hacer que parpadee, encenderlo con un botón y controlarlo con un botón mediante una condición if.

Para validar el funcionamiento, primero se realizaron pruebas en Wokwi, donde se analizaron tiempos de respuesta y compatibilidad. Luego, los códigos fueron cargados en los dispositivos físicos, midiendo su rendimiento en condiciones reales. Finalmente, se compararon aspectos como velocidad de ejecución, consumo de energía y facilidad de programación, permitiendo identificar las fortalezas y limitaciones de cada microcontrolador..

Simulación

Para evaluar el comportamiento de los microcontroladores antes de implementarlos físicamente, se utilizó la plataforma de simulación Wokwi. Esta herramienta permite probar códigos en un entorno virtual, replicando el funcionamiento real de dispositivos como el Arduino Uno, ESP32 y ATtiny.

En la simulación, se configuraron los circuitos con LEDs, resistencias y botones, asegurando que los cuatro programas funcionaran correctamente. Se analizaron aspectos como tiempos de respuesta, consumo de memoria y compatibilidad del código con cada microcontrolador. Además, Wokwi facilitó la depuración y ajuste de los programas antes de cargarlos en hardware real, optimizando el proceso de desarrollo y reduciendo posibles errores en la implementación física.

Diseño

El diseño del circuito se realizó de manera modular para adaptarse a los tres microcontroladores: Arduino Uno, ESP32 y ATtiny. Se usó una protoboard para facilitar las conexiones y permitir cambios rápidos entre dispositivos.

Cada circuito incluyó un LED con una resistencia de 220Ω, conectado a un pin de salida digital del microcontrolador. Para los ejercicios con botones, se agregaron pulsadores con resistencias de pull-down de 10kΩ, asegurando lecturas estables en las entradas digitales.
